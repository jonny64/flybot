flybot автоматизирует рутинные действия человека, использующего 
пиринговую файлообменную сеть DC, как-то: 
    * ответ на входящие личные сообщения (PM), 
    * добавление в игнор и закрытие PM рекламщиков (бесконечные 
      "новые хабы", рассылка "писем счастья"), 
    * автовыдача слота на скачивание при поступления PM с просьбой
    * не обрабатывает PM 'друзей'
входящие PM анализируются по шаблонам, задаваемыми регулярными 
выражениями (шаблоны в файле Settings\flydict.ini)
бот тестировался только на клиенте FlylinkDC версии r300-r394
домашняя страничка
http://flybot.googlecode.com

Небольшой FAQ
Q.: Где 'разбанить' пользователя?
A.: Добавление в игнор производится исключительно средствами клиента, 
соответственно, список забаненных следует искать там же, 
в окошке Favorites (Друзья).
 
Q.: Нужно ли мне знать регулярные выражения, чтобы создать правило 
для словаря?
A.: Не обязательно. Как шаблон входящей фразы может быть использована 
просто подстрока 'целевого' PM. Нужно будеть лишь предваярять спецсимволы бэкслешем.
То есть точка будет выглядеть так: \. слеш \/ и т.п. Синтаксис регулярных выражений описан тут: 
http://ru.wikipedia.org/wiki/Регулярные_выражения

Q.: Если подходят несколько шаблонов, какой будет выбран?
A.: Если входящее сообщение подходит под несколько шаблонов, для выбора того, который сработает,
 используются приоритеты. Выбор ответа происходит по принципу лотереи (у кого больше
билетов). Например, три правила с приоритетами 10, 20 и 50 разбивают отрезок на три
части с соотношением длин 1/10-1/20-1/50. На отрезок бросается (произвольно) 
точка. Сработает тот шаблон, в 'чью' часть отрезка попадет точка. 
То есть более высокий приоритет шаблона повышает вероятность его срабатывания.
